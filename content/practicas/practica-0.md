¿Qué es Markdown?
Markdown es un lenguaje de marcado ligero que permite dar formato a texto de manera sencilla. Fue creado para que los documentos sean fáciles de leer en texto plano y al mismo tiempo puedan convertirse en HTML u otros formatos.


¿Cómo se utiliza?
Markdown se utiliza escribiendo símbolos especiales en el texto que indican el formato, como encabezados, listas, enlaces o énfasis. Se usa comúnmente en documentación, blogs, README de proyectos y foros.


Sintaxis
•	Encabezado: # Título

•	Negrita: **texto**

•	Cursiva: *texto*

•	Lista no ordenada: - ítem

•	Lista ordenada: 1. ítem

•	Enlace: [texto](url)

•	Imagen: [alt](url)

•	Código en línea: `código`

¿Qué es Git y GitHub?

Git: Sistema de control de versiones que permite guardar y gestionar cambios en archivos de un proyecto. Ideal para trabajar de forma local y mantener un historial de modificaciones.

GitHub: Plataforma en la nube que almacena repositorios Git y permite colaborar con otras personas, compartir proyectos y controlar versiones de manera remota.

¿Cómo se utiliza?

Git se utiliza mediante línea de comandos o interfaces gráficas. 

GitHub se utiliza para subir repositorios, colaborar en proyectos y sincronizar los cambios entre equipos.

Comandos esenciales:
•	git config --global user.name "TuNombre" – Configurar nombre de usuario

•	git config --global user.email "tu@email.com" – Configurar email

•	git init – Inicializar repositorio local

•	git status – Ver estado de archivos

•	git add archivo.txt – Añadir archivo al área de preparación

•	git add . – Añadir todos los archivos

•	git commit -m "Mensaje descriptivo" – Guardar cambios (commit)

•	git log – Ver historial de commits

•	git remote add origin URL_DEL_REPOSITORIO – Conectar repositorio local con GitHub

•	git push -u origin main – Subir cambios al repositorio remoto

•	git clone URL_DEL_REPOSITORIO – Clonar un repositorio remoto

•	git checkout -b nombre_rama – Crear y cambiar a una rama nueva

•	git checkout nombre_rama – Cambiar a otra rama

•	git merge nombre_rama – Fusionar ramas

•	git pull origin main – Actualizar repositorio local desde remoto

¿Qué es Hugo y GitHub Actions?

Hugo: Generador de sitios estáticos escrito en Go. Permite crear páginas web rápidas a partir de archivos Markdown, plantillas y contenido estructurado.

GitHub Actions: Servicio de integración continua que permite automatizar tareas en GitHub, como compilar y desplegar un sitio web automáticamente al hacer cambios en el repositorio.
